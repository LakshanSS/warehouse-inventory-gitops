# Warehouse Inventory GitOps Repository

GitOps manifests for the warehouse inventory demo.

## Structure

```
warehouse-inventory-gitops/
├── platform/
│   ├── component-types/
│   │   ├── service.yaml              # Service component type
│   │   └── web-application.yaml      # Web app component type
│   └── infrastructure/
│       ├── environments/
│       │   └── development.yaml      # Dev environment
│       └── deployment-pipelines/
│           └── simple-pipeline.yaml  # Deployment pipeline
└── projects/
    └── default/
        ├── project.yaml
        └── components/
            ├── inventory-api-gitops/
            │   ├── component.yaml
            │   ├── workload.yaml     # Generated by workflow
            │   └── releases/         # Generated by workflow
            └── inventory-dashboard-gitops/
                ├── component.yaml
                ├── workload.yaml     # Generated by workflow
                └── releases/         # Generated by workflow
```

## How It Works

1. Trigger build: `kubectl apply -f demo/gitops/inventory-api-gitops-run.yaml`
2. Workflow builds image, generates manifests, creates PR
3. Review and merge PR
4. OpenChoreo syncs and deploys

## Components

| Component | Type | Port |
|-----------|------|------|
| `inventory-api-gitops` | service | 9090 |
| `inventory-dashboard-gitops` | web-application | 3000 |
