# Warehouse Inventory GitOps Repository

GitOps manifests for the warehouse inventory demo.

## Structure

```
warehouse-inventory-gitops/
└── projects/
    └── default/
        ├── project.yaml
        ├── components/
        │   ├── inventory-api-gitops/
        │   │   ├── component.yaml
        │   │   ├── workload.yaml          # Generated by workflow
        │   │   └── releases/              # Generated by workflow
        │   │       └── *.yaml
        │   └── inventory-dashboard-gitops/
        │       ├── component.yaml
        │       ├── workload.yaml          # Generated by workflow
        │       └── releases/              # Generated by workflow
        │           └── *.yaml
        ├── environments/
        │   └── development/
        │       └── environment.yaml
        └── pipelines/
            └── simple-pipeline/
                └── pipeline.yaml
```

## How It Works

1. Trigger build: `kubectl apply -f demo/gitops/inventory-api-gitops-run.yaml`
2. Workflow builds image, generates manifests, creates PR
3. Review and merge PR
4. OpenChoreo syncs and deploys

## Components

| Component | Type | Port |
|-----------|------|------|
| `inventory-api-gitops` | deployment/service | 9090 |
| `inventory-dashboard-gitops` | deployment/web-application | 3000 |
